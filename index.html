What is this project?
¿Qué es este proyecto?
A fully documented blueprint of a PS5 browser exploit. Think of it like architectural blueprints for a building — every room is planned, the structure is correct, but two critical pages are intentionally blank because they require hardware analysis nobody on this project has done yet.
Un plano completamente documentado de un exploit del navegador de PS5. Como los planos de un edificio — cada habitación está planificada, la estructura es correcta, pero dos páginas críticas están en blanco porque requieren un análisis de hardware que nadie en este proyecto ha realizado todavía.
🚫
No JIT compiler on PS5
Sin compilador JIT en PS5
Many circulating "PS5 exploits" claim to attack the JIT compiler. The PS5 browser has JIT disabled entirely. There is no JIT process to attack. Any project claiming "DFG JIT type confusion" for PS5 is technically impossible by definition.
Muchos "exploits de PS5" en circulación afirman atacar el compilador JIT. El navegador de PS5 tiene el JIT completamente deshabilitado. No hay proceso JIT al que atacar. Cualquier proyecto que afirme "DFG JIT type confusion" para PS5 es técnicamente imposible por definición.
🔗
Code execution via ROP only
Ejecución de código solo por ROP
Instead of injecting new code, ROP (Return-Oriented Programming) chains together tiny snippets of code that already exist in the console's memory. Like building sentences from words cut out of existing books. This is the only viable method on PS5 — and it requires knowing the exact memory addresses of those snippets, which vary per firmware version.
En lugar de inyectar código nuevo, ROP (Programación Orientada a Retornos) encadena pequeños fragmentos de código que ya existen en la memoria de la consola. Como construir frases con palabras recortadas de libros existentes. Este es el único método viable en PS5 — y requiere conocer las direcciones exactas de esos fragmentos, que varían por versión de firmware.
🔑
The single missing key
La única pieza que falta
Everything in this project stops at the same wall: a file called offsets_1100.js where every critical memory address is set to zero. Finding real values requires extracting and reverse-engineering the actual PS5 firmware 11.00 binaries — something that needs existing hardware access and specialized tools.
Todo en este proyecto se detiene en la misma pared: un archivo llamado offsets_1100.js donde cada dirección de memoria crítica está a cero. Encontrar los valores reales requiere extraer e invertir la ingeniería de los binarios reales del firmware PS5 11.00 — algo que necesita acceso a hardware existente y herramientas especializadas.

Broken pieces — detail
Piezas rotas — detalle
kernel.js — _pipeRead8/Write8 are empty stubs
kernel.js — _pipeRead8/Write8 son stubs vacíos
These three methods are the entire kernel R/W primitive. The pipe trick requires knowing the pipe_buffer.buffer field offset in the FW 11.00 kernel struct. Without the decrypted kernel binary, any value here is invented. The rest of KernelExploit is logically correct but unreachable without these.
Estos tres métodos son toda la primitiva R/W del kernel. El truco del pipe requiere conocer el offset del campo pipe_buffer.buffer en la struct del kernel de FW 11.00. Sin el binario del kernel desencriptado, cualquier valor aquí es inventado. El resto de KernelExploit es lógicamente correcto pero inalcanzable sin esto.
kernel.js — _ropMmap() returns Int64(0)
kernel.js — _ropMmap() devuelve Int64(0)
SYS_MMAP=197, PROT_RW=3, MAP_ANON|PRIVATE=0x1002 are stable across firmware. The blocker is capturing rax after the syscall — requires a mov [mem], rax; ret gadget whose offset in libkernel_web.sprx is unknown without the binary.
SYS_MMAP=197, PROT_RW=3, MAP_ANON|PRIVATE=0x1002 son estables entre firmwares. El bloqueo es capturar rax tras el syscall — requiere un gadget mov [mem], rax; ret cuyo offset en libkernel_web.sprx es desconocido sin el binario.
loader.js — missing gadget offsets cause silent 0x0 fallback
loader.js — offsets de gadgets ausentes causan fallback silencioso a 0x0
OFFSETS.GADGET_MOV_MEM_RAX and GADGET_MOV_RDI_MEM are not in offsets_1100.js. The || 0 fallback means the ROP chain uses libkernelBase + 0x0 as a gadget address — the binary header, not a gadget — and execution immediately corrupts. These are real gadgets that exist in any large enough x86-64 binary; the offset just needs to be found with ROPgadget.
OFFSETS.GADGET_MOV_MEM_RAX y GADGET_MOV_RDI_MEM no están en offsets_1100.js. El fallback || 0 hace que la cadena ROP use libkernelBase + 0x0 como dirección de gadget — la cabecera del binario, no un gadget — y la ejecución se corrompe inmediatamente. Son gadgets reales que existen en cualquier binario x86-64 suficientemente grande; el offset solo necesita encontrarse con ROPgadget.
pt.c — spin-wait in a borrowed thread is dangerous
pt.c — spin-wait en un hilo prestado es peligroso
The injected shellcode runs in a borrowed thread from SceRedisServer. A busy-wait loop at 100% CPU can trigger Orbis OS's watchdog and kill the process. The correct fix is nanosleep() or sched_yield() from libkernel — both of which also have unknown offsets.
El shellcode inyectado corre en un hilo prestado de SceRedisServer. Un bucle de espera activa al 100% de CPU puede activar el watchdog de Orbis OS y matar el proceso. La corrección correcta es nanosleep() o sched_yield() de libkernel — ambos también con offsets desconocidos.
