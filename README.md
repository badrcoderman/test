---

## 📌 نظرة عامة (Overview)

تم بناء هذه الأداة لنقل الفحص الأمني من مرحلة **المراجعة النظرية للكود (Source Review)** إلى مرحلة **التحقق الفعلي والعملي (Bug Validation)** عبر بيئة اختبار رسومية محاطة بسجل تصحيح أخطاء (Debug Log) فوري ومباشر على الشاشة.

---

## 🛠️ الميزات الأساسية (Key Features)

* **💻 واجهة رسومية تفاعلية:** تصميم داكن مريح للعين يحاكي واجهات أنظمة الفحص المتقدمة.
* **⏱️ سجل فوري (Real-time Debug Log):** تتبع دقيق بالوقت الفعلي لكل عملية تتم داخل الذاكرة (إنشاء، تعديل، حذف).
* **🎯 محاكاة التداخل العنيف:** إجبار المحرك على استدعاء الـ Callbacks مع محاولة قطع سلسلة المراجع يدوياً عبر الـ JavaScript.
* **🎨 ترميز لوني للأخطاء:** تمييز الحالات (Info, Success, Warning, Error) لتسهيل القراءة أثناء المراقبة.

## 🚀 طريقة التشغيل والاستخدام (How to Use)

1. **تحميل المشروع:** قم بتحميل أو نسخ ملف `index.html` إلى جهازك أو بيئة المحاكاة لديك.
2. **التشغيل:** افتح الملف باستخدام أي متصفح ويب مرتبط ببيئة التصحيح (مثل متصفح مدمج مع أدوات مثل **ASan** أو **UBSan**).
3. **بدء الفحص:** اضغط على زر **"بدء الفحص والاختبار"** لمراقبة تتابع الأحداث.

---

## 🔍 قراءة وتحليل النتائج (Log Analysis)

* **الحالة المستقرة (Safe Posture):** إذا استمرت الصفحة في العمل وظهرت عبارة `[SUCCESS] تم دورة الاختبار بنجاح`، فهذا يعني أن المحرك يعالج الـ Lifetime الخاص بالكائنات بشكل آمن تماماً عبر المؤشرات الذكية.
* **حالة الانهيار (Crash State):** إذا توقفت الصفحة فجأة عن طباعة السطور أو انهار التبويب بالكامل، فهذا مؤشر على نجاح محاكاة الخلل بالذاكرة، ويجب فحص مخرجات الـ **ASan** في الطرفية (Terminal) لمعاينة التقرير.
